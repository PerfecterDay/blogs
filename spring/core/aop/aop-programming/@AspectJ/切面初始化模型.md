# 切面的初始化模型
{docsify-updated}
> https://docs.spring.io/spring-framework/reference/core/aop/ataspectj/instantiation-models.html

默认情况下，应用程序上下文中每个切面仅存在单个实例。AspectJ 将此称为单例实例化模型。用户可定义具有其它生命周期的切面。Spring 支持 AspectJ 的 `perthis` 、 `pertarget` 和 `pertypewithin` 实例化模型；目前尚不支持 `percflow` 和 `percflowbelow` 模型。

可以在 `@Aspect` 注解中指定 `perthis` 子句来声明一个 `perthis` 切面。请看以下示例：
```
@Aspect("perthis(execution(* com.xyz..service.*.*(..)))")
public class MyAspect {
	private int someState;

	@Before("execution(* com.xyz..service.*.*(..))")
	public void recordServiceUsage() {
		// ...
	}
}
```
在上例中， `perthis` 子句的作用是为每个执行业务服务的对象创建一个切面实例（每个唯一对象在切入点表达式匹配的连接点处与该切面绑定）。当服务对象的方法首次被调用时，切面实例即被创建。当服务对象超出作用域时，切面也随之超出作用域。在切面实例创建之前，其内部的所有增强都不会执行。一旦切面实例创建完成，其中声明的增强就会在匹配的连接点上运行，但仅当服务对象是与该切面关联的对象时才生效。

`pertarget` 实例化模型的工作原理与 `perthis` 完全相同，但它会在匹配的连接点处为每个唯一的目标对象创建一个切面实例。

`perthis` 是为每个连接点匹配的对象创建一个实例，比如一个对象中有三个方法都匹配到了切面，那么 `perthis` 只会为该对象创建一个切面实例。 但是 `pertarget` 会创建三个实例。
